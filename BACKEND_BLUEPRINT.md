# VibeShopping - Backend Blueprint

> Complete backend architecture aligned with the VibeShopping mobile UI (Paper MCP).
> Generated from analysis of 7 UI screens: Login, Signup, Home, Product List, Categories, Cart, Profile.

---

## 1. UI Screen вҶ’ Backend Feature Mapping

| UI Screen | UI Actions | Backend API Required |
|---|---|---|
| **Login** | Email/phone + password login, Google/Apple OAuth, Forgot Password | `POST /auth/login`, `POST /auth/social`, `POST /auth/forgot-password` |
| **Signup** | Register with name, email, phone, password | `POST /auth/register` |
| **Home** | Greeting (user name), search products, view flash sale banner, browse categories, view featured products, wishlist toggle | `GET /auth/me`, `GET /products/search`, `GET /banners`, `GET /categories`, `GET /products/featured`, `POST /wishlist/toggle` |
| **Product List** | Filter by category, sort (price/rating/new), view product grid, wishlist toggle | `GET /categories/{id}/products?sort=&filters=` |
| **Categories** | Browse all categories with product counts | `GET /categories` |
| **Cart** | View cart items, update quantity, remove item, view price summary, proceed to checkout | `GET /cart`, `POST /cart/items`, `PUT /cart/items/{id}`, `DELETE /cart/items/{id}` |
| **Profile** | View/edit profile, My Orders, Addresses, Payment Methods, Settings, Logout | `GET/PUT /users/profile`, `GET /orders`, `GET/POST/PUT/DELETE /addresses`, `GET/POST/DELETE /payment-methods`, `POST /auth/logout` |

## 2. Tech Stack

| Layer | Technology |
|---|---|
| **Framework** | Python 3.12 + FastAPI |
| **Database** | PostgreSQL 16 |
| **ORM** | SQLAlchemy 2.0 (async) |
| **Migrations** | Alembic |
| **Auth** | JWT (access + refresh tokens) |
| **Validation** | Pydantic V2 |
| **Storage** | AWS S3 (product images, avatars) |
| **Cache** | Redis (sessions, cart, search) |
| **Task Queue** | Celery + Redis (emails, notifications) |
| **Testing** | pytest + httpx |
| **API Docs** | OpenAPI 3.1 (auto-generated by FastAPI) |

## 4. API Endpoints (Complete)

### Standard Response Envelope

```json
{
  "success": true,
  "data": {},
  "message": "string"
}
```

Paginated responses add:
```json
{
  "success": true,
  "data": [],
  "message": "string",
  "pagination": {
    "page": 1,
    "page_size": 20,
    "total": 100,
    "total_pages": 5
  }
}
```

### 4.1 Auth Module

| Method | Endpoint | Description | Auth | UI Source |
|---|---|---|---|---|
| POST | `/api/v1/auth/register` | Create account | No | Signup screen |
| POST | `/api/v1/auth/login` | Email/phone + password login | No | Login screen |
| POST | `/api/v1/auth/social` | Google/Apple OAuth | No | Login social buttons |
| POST | `/api/v1/auth/refresh` | Refresh access token | Refresh token | Auto |
| POST | `/api/v1/auth/forgot-password` | Send reset email | No | Login "Forgot Password?" |
| POST | `/api/v1/auth/reset-password` | Reset with token | No | Email link |
| POST | `/api/v1/auth/logout` | Invalidate tokens | JWT | Profile "Logout" |
| GET | `/api/v1/auth/me` | Get current user | JWT | Home greeting "Good morning, Akhil" |

### 4.2 Users Module

| Method | Endpoint | Description | Auth | UI Source |
|---|---|---|---|---|
| GET | `/api/v1/users/profile` | Get full profile | JWT | Profile screen header |
| PUT | `/api/v1/users/profile` | Update name/email/phone/avatar | JWT | Profile вҶ’ Settings |

### 4.3 Categories Module

| Method | Endpoint | Description | Auth | UI Source |
|---|---|---|---|---|
| GET | `/api/v1/categories` | List all categories (with product counts) | No | Categories screen, Home categories row |
| GET | `/api/v1/categories/{id}` | Get single category | No | Category detail |
| GET | `/api/v1/categories/{id}/products` | Products in category (paginated, sortable, filterable) | No | Product List screen |

**Query params for category products:**
- `sort`: `price_asc`, `price_desc`, `rating`, `newest` (from Product List filter chips)
- `min_rating`: float (from "Rating 4+" filter)
- `min_price` / `max_price`: decimal (from "Price: Low-High")
- `is_new`: boolean (from "New" filter)
- `page` / `page_size`: pagination

### 4.4 Products Module

| Method | Endpoint | Description | Auth | UI Source |
|---|---|---|---|---|
| GET | `/api/v1/products` | List all products (paginated) | No | General browsing |
| GET | `/api/v1/products/featured` | Featured products | No | Home "Featured" section |
| GET | `/api/v1/products/search?q=` | Full-text search | No | Home search bar |
| GET | `/api/v1/products/{id}` | Product detail (with variants, images) | No | Product card tap (implied) |
| GET | `/api/v1/products/{id}/reviews` | Reviews for product | No | Product detail (implied) |

### 4.5 Wishlist Module

| Method | Endpoint | Description | Auth | UI Source |
|---|---|---|---|---|
| GET | `/api/v1/wishlist` | List wishlisted products | JWT | Implied from heart icons |
| POST | `/api/v1/wishlist/toggle` | Add/remove from wishlist | JWT | Heart icon on product cards |

**Request body:** `{ "product_id": "uuid" }`

### 4.6 Cart Module

| Method | Endpoint | Description | Auth | UI Source |
|---|---|---|---|---|
| GET | `/api/v1/cart` | Get cart (items + price summary) | JWT | Cart screen |
| POST | `/api/v1/cart/items` | Add item to cart | JWT | Product detail "Add to Cart" |
| PUT | `/api/v1/cart/items/{id}` | Update quantity | JWT | Cart quantity +/- buttons |
| DELETE | `/api/v1/cart/items/{id}` | Remove item | JWT | Cart item swipe/delete |

**Cart response includes computed fields:** subtotal, delivery_fee, total (matching Cart "Price Summary" section).

### 4.7 Orders Module

### 4.8 Payments Module

### 4.9 Addresses Module

| Method | Endpoint | Description | Auth | UI Source |

### 4.10 Payment Methods Module

### 4.11 Reviews Module

| Method | Endpoint | Description | Auth | UI Source |

### 4.12 Banners Module

| Method | Endpoint | Description | Auth | UI Source |

### 7.3 Repository вҖ” `app/repositories/product_repo.py`

```python
from __future__ import annotations

from uuid import UUID

from sqlalchemy import func, select
from sqlalchemy.ext.asyncio import AsyncSession

from app.models.product import Product
from app.models.wishlist import Wishlist
from app.schemas.product import ProductSearchParams


class ProductRepository:
    def __init__(self, db: AsyncSession):
        self.db = db

    async def get_by_id(self, product_id: UUID) -> Product | None:
        result = await self.db.execute(
            select(Product).where(Product.id == product_id, Product.is_active.is_(True))
        )
        return result.scalar_one_or_none()

    async def get_featured(self, limit: int = 10) -> list[Product]:
        result = await self.db.execute(
            select(Product)
            .where(Product.is_featured.is_(True), Product.is_active.is_(True))
            .order_by(Product.created_at.desc())
            .limit(limit)
        )
        return list(result.scalars().all())

    async def search(
        self, params: ProductSearchParams, category_id: UUID | None = None
    ) -> tuple[list[Product], int]:
        query = select(Product).where(Product.is_active.is_(True))
        count_query = select(func.count(Product.id)).where(Product.is_active.is_(True))

        if category_id:
            query = query.where(Product.category_id == category_id)
            count_query = count_query.where(Product.category_id == category_id)

        if params.q:
            query = query.where(Product.name.ilike(f"%{params.q}%"))
            count_query = count_query.where(Product.name.ilike(f"%{params.q}%"))

        if params.min_rating:
            query = query.where(Product.avg_rating >= params.min_rating)
            count_query = count_query.where(Product.avg_rating >= params.min_rating)

        if params.min_price:
            query = query.where(Product.base_price >= params.min_price)
            count_query = count_query.where(Product.base_price >= params.min_price)

        if params.max_price:
            query = query.where(Product.base_price <= params.max_price)
            count_query = count_query.where(Product.base_price <= params.max_price)

        # Sorting (maps to Product List filter chips)
        sort_map = {
            "price_asc": Product.base_price.asc(),
            "price_desc": Product.base_price.desc(),
            "rating": Product.avg_rating.desc(),
            "newest": Product.created_at.desc(),
        }
        query = query.order_by(sort_map.get(params.sort, Product.created_at.desc()))

        # Pagination
        offset = (params.page - 1) * params.page_size
        query = query.offset(offset).limit(params.page_size)

        total = (await self.db.execute(count_query)).scalar() or 0
        products = list((await self.db.execute(query)).scalars().all())

        return products, total

    async def get_wishlisted_ids(self, user_id: UUID) -> set[UUID]:
        result = await self.db.execute(
            select(Wishlist.product_id).where(Wishlist.user_id == user_id)
        )
        return set(result.scalars().all())
```

### 7.4 Service вҖ” `app/services/product_service.py`

```python
from __future__ import annotations

from uuid import UUID

from sqlalchemy.ext.asyncio import AsyncSession

from app.repositories.product_repo import ProductRepository
from app.schemas.product import ProductDetail, ProductListItem, ProductSearchParams


class ProductService:
    def __init__(self, db: AsyncSession):
        self.repo = ProductRepository(db)

    async def get_featured(
        self, user_id: UUID | None = None
    ) -> list[ProductListItem]:
        products = await self.repo.get_featured()
        wishlisted = await self.repo.get_wishlisted_ids(user_id) if user_id else set()
        return [self._to_list_item(p, wishlisted) for p in products]

    async def search(
        self,
        params: ProductSearchParams,
        category_id: UUID | None = None,
        user_id: UUID | None = None,
    ) -> tuple[list[ProductListItem], int]:
        products, total = await self.repo.search(params, category_id)
        wishlisted = await self.repo.get_wishlisted_ids(user_id) if user_id else set()
        items = [self._to_list_item(p, wishlisted) for p in products]
        return items, total

    async def get_detail(
        self, product_id: UUID, user_id: UUID | None = None
    ) -> ProductDetail | None:
        product = await self.repo.get_by_id(product_id)
        if not product:
            return None
        wishlisted = await self.repo.get_wishlisted_ids(user_id) if user_id else set()
        return ProductDetail.model_validate(
            product, update={"is_wishlisted": product.id in wishlisted}
        )

    @staticmethod
    def _to_list_item(product, wishlisted: set[UUID]) -> ProductListItem:
        primary_img = next(
            (img.image_url for img in product.images if img.is_primary), None
        )
        return ProductListItem(
            id=product.id,
            name=product.name,
            base_price=product.base_price,
            avg_rating=product.avg_rating,
            review_count=product.review_count,
            primary_image_url=primary_img,
            is_wishlisted=product.id in wishlisted,
        )
```

```python
from app.core.responses import ApiResponse, paginated_response
from app.schemas.product import ProductSearchParams
from app.services.product_service import ProductService

router = APIRouter(prefix="/products", tags=["Products"])


@router.get("")
async def list_products(
    q: str | None = Query(None),
    sort: str = Query("newest"),
    min_rating: float | None = Query(None),
    page: int = Query(1, ge=1),
    page_size: int = Query(20, ge=1, le=100),
    db: AsyncSession = Depends(get_db),
    user_id: UUID | None = Depends(get_current_user_optional),
):
    service = ProductService(db)
    params = ProductSearchParams(
        q=q, sort=sort, min_rating=min_rating, page=page, page_size=page_size
    )
    items, total = await service.search(params, user_id=user_id)
    return paginated_response(items, page, page_size, total)

async def featured_products(
    db: AsyncSession = Depends(get_db),
    user_id: UUID | None = Depends(get_current_user_optional),
):
    service = ProductService(db)
    items = await service.get_featured(user_id)
    return ApiResponse(success=True, data=items, message="Featured products")

    q: str = Query(..., min_length=1),

    service = ProductService(db)
    params = ProductSearchParams(q=q, sort=sort, page=page, page_size=page_size)
    items, total = await service.search(params, user_id=user_id)

async def get_product(
    product_id: UUID,
    db: AsyncSession = Depends(get_db),
    user_id: UUID | None = Depends(get_current_user_optional),
):

    product = await service.get_detail(product_id, user_id)
```

### 7.6 Response Helpers вҖ” `app/core/responses.py`

```python
from __future__ import annotations

from typing import Any

from pydantic import BaseModel


class ApiResponse(BaseModel):
    success: bool
    data: Any
    message: str


class PaginationMeta(BaseModel):
    page: int
    page_size: int
    total: int
    total_pages: int


class PaginatedResponse(BaseModel):
    success: bool
    data: Any
    message: str
    pagination: PaginationMeta


def paginated_response(
    items: list, page: int, page_size: int, total: int, message: str = "OK"
) -> dict:
    return PaginatedResponse(
        success=True,
        data=items,
        message=message,
        pagination=PaginationMeta(
            page=page,
            page_size=page_size,
            total=total,
            total_pages=(total + page_size - 1) // page_size,
        ),
    ).model_dump()
```

## 9. Key Design Decisions

| Decision | Rationale |
|---|---|
| **Product variants** | Cart shows "Color: Black", "Size: M" вҖ” products need variant support |
| **Order item snapshots** | `order_items` stores `product_name` and `unit_price` at time of purchase вҖ” prices may change |
| **Wishlist as toggle** | UI shows heart icons that toggle вҖ” single `POST /wishlist/toggle` is simpler than separate add/remove |
| **Optional auth on product endpoints** | Product browsing works without login; wishlisted state is enriched when authenticated |
| **Banner entity** | Home screen has "FLASH SALE" with title, subtitle, CTA вҖ” needs a manageable content model |
| **Refresh token rotation** | Security best practice вҖ” each refresh invalidates the previous token |
| **pg_trgm for search** | Home search bar needs fuzzy product name matching вҖ” trigram index is simple and effective |
